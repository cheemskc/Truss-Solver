# Matrix Structural Solver (矩阵位移法结构求解引擎)

基于 Python 开发的二维杆系结构矩阵位移法求解引擎。本项目提供从有限元模型建立、刚度矩阵组装、线性方程组求解到杆件内力提取的完整计算流程，适用于二维桁架与刚架结构的静力学分析。

---

## 计算原理 (Calculation Principles)

本项目核心算法基于经典的**直接刚度法 (Direct Stiffness Method)**，底层计算流程严格遵循有限元分析步骤：

1. **局部刚度矩阵建立**: 
   对于每一个二维梁/杆单元，根据其材料弹性模量 $E$、截面面积 $A$、截面惯性矩 $I$ 以及单元长度 $L$，建立 $6 \times 6$ 的局部坐标系单元刚度矩阵 $[k^e]$。对于只受轴力的桁架，可将 $EI$ 设为极小值。
2. **坐标变换**: 
   计算单元从局部坐标系到全局坐标系的转换矩阵 $[T]$，得到全局坐标系下的单元刚度矩阵：
   $$[K^e] = [T]^T [k^e] [T]$$
3. **全局刚度矩阵组装**: 
   根据单元与节点的拓扑连接关系（自由度映射），将所有单元的全局刚度矩阵 $[K^e]$ 叠加，组装成总结构刚度矩阵 $[K]$。
4. **荷载向量组装**: 
   将节点集中荷载与单元的固端力 (Fixed-End Forces) 转化为等效节点荷载，合并生成结构的总荷载向量 $\{P\}$。
5. **引入边界条件**: 
   采用对角线乘大数法 (Penalty Method) 或划零置一法处理支座约束，修正 $[K]$ 和 $\{P\}$。
6. **求解位移与内力**: 
   求解线性代数方程组：
   $$[K] \{U\} = \{P\}$$
   解出所有节点的全局位移 $\{U\}$。随后提取各单元的节点位移，通过 $[T]$ 转换回局部坐标系，并结合初始固端力计算出单元的最终局部内力。

## 计算建模注意事项 (Calculation Caveats)

在使用本求解器构建力学模型时，请务必严格遵守以下计算边界与符号约定，这是有限元计算中最容易出错的环节：

### 1. 物理单位的统一性 (Unit Consistency)
求解器底层矩阵运算是“无量纲”的，程序不会自动转换单位。输入数据的物理单位必须在外部保持绝对统一。
* **推荐单位制 (SI/kN-m)**：长度统一使用 $m$，力使用 $kN$。
* 对应的刚度输入应为：$EA$ 取 $kN$，$EI$ 取 $kN\cdot m^2$。
* 集中力输入取 $kN$，力矩取 $kN\cdot m$。
* 计算输出的结果将严格对应：线位移为 $m$，角位移为 $rad$。

### 2. 局部坐标系与正负号约定 (Sign Conventions)
* **全局坐标系**: $X$ 轴水平向右为正，$Y$ 轴竖直向上为正，力矩与转角均以**逆时针**方向为正。
* **局部坐标系**: 单元的局部 $x$ 轴始终**从起点 (Node i) 指向终点 (Node j)**。局部 $y$ 轴由局部 $x$ 轴逆时针旋转 $90^\circ$ 得到。
* **内力正负号**: 输出的内力结果严格遵循单元局部坐标系。例如，轴力 $N$ 为正代表截面受拉；剪力 $V$ 与弯矩 $M$ 的正负依据经典的材料力学局部截面符号规定。

### 3. 刚体位移与奇异矩阵 (Singular Matrix)
如果点击求解后程序报错“矩阵奇异”或输出的位移值异常巨大（例如达到 $10^{10}$ 量级），通常意味着刚度矩阵 $[K]$ 不可逆。可能的原因包括：
* **约束不足**: 结构整体缺乏足够的支座约束，导致存在沿 $X$、$Y$ 或旋转方向的刚体位移 (Rigid body motion)。
* **几何可变体系**: 单元拓扑连接属于机构 (Mechanism)，内部存在不受约束的局部自由度。
* **极度悬殊的刚度**: 输入的 $EA$ 与 $EI$ 数值差异过大（例如相差 $10^{16}$ 以上），导致病态矩阵，引发浮点数舍入误差。

### 4. 等效节点荷载与固端力 (Fixed-End Forces)
本求解器主要处理节点集中荷载。若模型中存在非节点荷载（如杆件跨中均布荷载、集中力），需由用户根据结构力学图表，先手工将跨中荷载转化为起止点的**固端力 (FEF)**，并填入“单元属性矩阵”中。
* **注意**: 输入的 FEF 值必须是杆件对两端节点的**固端反力**，方向遵循局部坐标系的正负规定。
